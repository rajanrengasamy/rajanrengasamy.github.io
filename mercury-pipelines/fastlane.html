Pipeline Overview

The Fast Lane is Mercury's continuous content engine. It monitors multiple signal sources, filters and prioritizes content opportunities, drafts posts in Rajan's voice, and publishes up to 8 times per day through automated X API integration.

Signal Sources

  • X Fast Lane — Trending AI/tech topics from X API
  • Newsletter Scanner — Curated AI newsletters (Ben's Bites, etc.)
  • YouTube RSS — @AIJasonZ, Nick Saraev, other AI channels
  • Reddit Monitor — r/LocalLLaMA, r/ClaudeAI, r/OpenAI
  • Hacker News — Top AI/tech stories
  • GitHub Trending — New AI repos and tools
  • ARC Builds — Overnight R&D project outputs

Quality Gates

  • Gate 0 (Slop Test) — Would someone call this AI slop?
  • Gate 1 (Voice) — Matches Rajan's builder voice, no em-dashes
  • Gate 2 (Freshness) — Not stale (>6 days = reject)
  • Gate 3 (Differentiation) — Unique angle, not generic
  • Uday Test — Starts with lowercase or number

Stage Details

1. Sensors

Collect raw signals from 7+ sources every 30 minutes. No LLM needed — pure data collection.

No LLMInput

2. Dedup + Priority

Hash-based deduplication and priority scoring. Prevents duplicate posts and ranks opportunities.

Rules + SQLiteFilter

3. LLM Triage

Classify each signal and assign action: draft, post_now, defer, log, or ignore.

Haiku 4Decision

4. Draft Generation

Generate tweet content in Rajan's voice with source attribution and thread structure.

Sonnet 4.5Creation

5. OracleX Gate

Final editorial review against 5 quality gates. Kill if any gate fails.

Opus 4.6Quality

6. X API Posting

Post via Python script with rate limiting. Supports standalone + thread replies.
