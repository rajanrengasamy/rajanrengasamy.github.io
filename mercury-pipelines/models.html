Model Routing Matrix

How Mercury routes tasks to the right LLM for cost, speed, and quality

Design Principles

  • Mercury is always Opus 4.6 — Editorial judgment, strategy, OracleX review require top-tier reasoning
  • Fast classification uses Haiku — 500+ signals/day need speed and low cost
  • Content generation uses Sonnet — Balanced quality/speed for writing
  • Build uses GLM-5 or Codex — Code generation specialist models
  • $0 API cost — All covered by subscriptions (Claude Max, GPT Pro)

Routing Logic

openclaw_bridge.js routing:
1. Check if Anthropic OAuth token available
2. If yes → direct API call (fastest)
3. If no → CLI fallback via OpenClaw → GPT-5.4
4. Claude Max OAuth (sk-ant-oat) forces CLI route

The bridge code explicitly prefers CLI fallback when using Claude Max OAuth tokens, since raw Anthropic API doesn't accept OAuth tokens. This routes through the Gateway to the configured default model (currently GPT-5.4).
